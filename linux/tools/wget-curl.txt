wget命令用来从指定的URL下载文件。
wget非常稳定，它在带宽很窄的情况下和不稳定网络中有很强的适应性，
如果是由于网络的原因下载失败，wget会不断的尝试，直到整个文件下载完毕。
如果是服务器打断下载过程，它会再次联到服务器上从停止的地方继续下载。
这对从那些限定了链接时间的服务器上下载大文件非常有用。

wget (选项) (参数)

-b          后台执行
-o (file)   log messages to FILE.
-a (file)   append messages to FILE

-t (num)    set number of retries to NUMBER (0 unlimits)
-O (file)   write documents to FILE.
-c          resume getting a partially-downloaded file.
-S          print server response.
-w (seconds) wait SECONDS between retrievals.

wget http://www.linuxde.net/testfile.zip

curl
它支持文件的上传和下载，所以是综合传输工具，但按传统，习惯称curl为下载工具。
作为一款强力工具，curl支持包括HTTP、HTTPS、ftp等众多协议，
还支持POST、cookies、认证、从指定偏移处下载部分文件、用户代理字符串、限速、文件大小、进度条等特征。
做网页处理流程和数据检索自动化，curl可以祝一臂之力。

-A, --user-agent            | STRING  User-Agent to send to server
-u, --user USER[:PASSWORD]  | Server user and password (完成HTTP或者FTP的认证)
-T, --upload-file (FILE)    | Transfer FILE to destination
-o, --output FILE           | Write output to <file> instead of stdout
-H, --header LINE           | Custom header to pass to server (H)
-I, --head                  | Show document info only
-d, --data DATA             | HTTP POST data (H)

不显示进度信息使用 --silent
-O      将下载的数据写入到文件，必须使用文件的绝对地址
-o      将下载数据写入到指定名称的文件中，并使用 --progress 显示进度条：
curl http://man.linuxde.net/text.iso --silent -O
curl http://man.linuxde.net/test.iso -o filename.iso --progress


curl -H "Content-Type:application/json" -X POST --data '{"message": "sunshine"}' http://localhost:8000/
